# 联系管理员（dialogue）

用于让用户直接给机器人管理员留言，并支持管理员快速回信。

## 用户怎么用

发送：

```bash
滴滴滴- 你的留言内容
```

例如：

```bash
滴滴滴- 管理员你好，我这边抽卡好像有问题
```

说明：

- 支持文本，也可附带图片一起发送。
- 发送后，机器人会把你的昵称、用户 ID、群号（如果在群里）和消息内容转发给管理员。

## 管理员怎么用（超级用户）

### 1. 分页查看消息

```bash
# 查看未回复（默认）
/t list

# 查看全部
/t list all

# 查看第 2 页，每页 20 条
/t list all 2 20

# 显式查看未回复
/t list pending 1 10
```

说明：

- 支持两种模式：`all`（全部）和 `pending`（仅未回复，默认）。
- 分页参数：`[页码] [每页数量]`，每页数量最大 50。
- 列表中会显示 `Id`，用于后续按 `Id` 回复。

### 2. 按留言 Id 回复（推荐）

```bash
/t <id> <回复内容>
```

例如：

```bash
/t 0 收到啦，我们会尽快处理
```

说明：

- 这里的 `id` 是 `/t` 列表中显示的 `Id`。
- 按 `id` 回复后，会把回复内容写入数据库并标记为已回复。
- 如果原消息来自群聊，会在原群里 @对应用户回复；来自私聊则私聊回复。

### 3. 指定用户/群发送

```bash
# 私聊指定用户
/t <user_id> <文本>

# 在群里 @指定用户回复
/t <user_id> <group_id> <文本>

# 只在群里发消息（不@用户）
/t -1 <group_id> <文本>
```

例如：

```bash
/t 123456789 你好，这边帮你看了
/t 123456789 987654321 问题已修复
/t -1 987654321 请大家注意文明发言
```

## 注意事项

- 该插件的待回复消息现在保存在数据库中，机器人重启后不会丢失。
- 如果提示“管理员失联了...”，说明当前平台没有可用超级用户配置。
