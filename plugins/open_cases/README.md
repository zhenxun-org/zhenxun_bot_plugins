# open_cases（CSGO 开箱模拟）

一个群聊内可用的 CSGO 开箱模拟插件，支持单开/连开、个人统计、群统计、金色记录和价格趋势查看。

## 功能一览

- 模拟开箱（随机武器箱或指定武器箱）
- 连开箱（一次最多 30 次）
- 查看个人开箱统计
- 查看个人金色掉落记录
- 查看群组总统计
- 查看武器箱内容
- 查看皮肤价格趋势
- 超级用户可手动更新武器箱/皮肤数据

## 快速开始

1. 登录 [BUFF](https://buff.163.com/)。
2. 浏览器按 `F12` 打开开发者工具，在网络请求中复制 `Cookie`。
3. 将 Cookie 填到插件配置 `COOKIE`（建议放在 `data/config.yaml` 对应插件项）。
4. 重载配置或重启 Bot。
5. 超级用户先执行一次：`更新武器箱 ALL`。

## 用户命令

- `开箱 [武器箱名]`
- `[1-30]连开箱 [武器箱名]`
- `我的开箱`
- `我的金色`
- `群开箱统计`
- `查看武器箱 [武器箱名]`
- `价格趋势 <武器箱名> <武器名> <磨损> [天数]`

说明：
- 不写武器箱名时，会随机开箱。
- `价格趋势` 的天数范围是 `1-180`，不填默认 `7` 天。

## 超级用户命令

- `更新武器箱 [武器箱名/ALL]`
- `更新皮肤 [名称/ALL1]`
- `重置开箱`（重置当日开箱次数）

## 规则与限制

- 连开箱次数必须 `> 0` 且 `<= 30`。
- 每日开箱上限：
  - `INITIAL_OPEN_CASE_COUNT + impression / EACH_IMPRESSION_ADD_COUNT`
- 每日会自动重置开箱次数（定时任务）。
- 配置了 `DAILY_UPDATE` 后，会在每日定时自动更新对应武器箱数据。

## 配置项

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `INITIAL_OPEN_CASE_COUNT` | `20` | 每日基础开箱次数 |
| `EACH_IMPRESSION_ADD_COUNT` | `3` | 每多少好感度增加 1 次开箱次数 |
| `COOKIE` | `None` | BUFF Cookie（必填，建议定期更新） |
| `DAILY_UPDATE` | `None` | 每日自动更新的武器箱列表，包含 `ALL` 时更新全部 |
| `BUFF_PROXY` | `None` | BUFF 请求代理地址 |
| `DEFAULT_OPEN_CASE_RESET_REMIND` | `True` | 每日开箱重置提醒任务默认开关 |

## 常见问题

- 提示“武器箱未收录”：
  - 先执行 `更新武器箱`（或 `更新武器箱 ALL`）同步数据。
- 提示需要登录或更新失败：
  - 一般是 `COOKIE` 失效，重新抓取后覆盖配置并重启。
- 提示开箱次数不足：
  - 等次日重置，或提升好感度获得更多次数。

## 注意事项

- BUFF 请求过于频繁可能触发风控，请不要高频连续更新全部数据。
- 建议每天最多进行 1 次全量更新，其他时间按需更新单个武器箱/皮肤。
